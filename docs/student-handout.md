## Практика 15 — HTTPS + App Shell + кеширование

### Цель

1) Поднять проект по **HTTPS** локально (иначе SW/Push работают ограниченно).
2) Перейти к подходу **App Shell**:
   - оболочка приложения (шапка/навигация/контейнер) — в `index.html`
   - контент страниц — в папке `content/`
3) Настроить кеширование так, чтобы:
   - оболочка и статика работали офлайн
   - страницы из `content/` подгружались и кешировались

### Что уже сделано в заготовке

- В `index.html` есть контейнер для контента.
- В `service-worker.js` есть базовая структура кеширования (и TODO).

## Как проверяем (к сдаче)

1) Открыть приложение **по HTTPS**.
2) Проверить, что при перезагрузке часть ресурсов доступна из кеша.
3) WebSocket:
   - открыть приложение в двух вкладках
   - создать задачу в первой → во второй увидеть обновление
4) Push:
   - подписаться
   - отправить push с сервера → получить уведомление
